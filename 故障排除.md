# JLPT N1 学习网站 - 问题诊断和解决方案

## 当前问题

Vite开发服务器启动后立即退出,无法正常启动。

## 诊断步骤

### 1. 运行诊断脚本
双击 `start.bat` 文件,它会显示:
- 当前目录
- Node版本
- NPM版本  
- 安装过程
- 启动错误信息

### 2. 检查环境要求

**必需:**
- Node.js >= 18.0.0 (推荐 20.x)
- npm >= 9.0.0

**当前版本:**
```bash
node --version   # 应该是 v18+ 或 v20+
npm --version    # 应该是 9+
```

## 解决方案

### 方案1: 升级Node.js (推荐)

你的Node版本是 v22.12.0,但某些包需要 v22.13.0+。

1. 下载最新LTS版本: https://nodejs.org/
2. 安装后重启电脑
3. 重新运行 `start.bat`

### 方案2: 使用Node 20 LTS

1. 使用nvm-windows管理Node版本:
   ```bash
   # 安装nvm-windows
   # 从 https://github.com/coreybutler/nvm-windows/releases 下载
   
   # 安装Node 20
   nvm install 20
   nvm use 20
   
   # 进入项目目录
   cd "c:\Users\user17\Desktop\新建文件夹\日语\git words\jlpt-n1-study-new"
   
   # 删除现有依赖
   Remove-Item -Recurse -Force node_modules
   Remove-Item package-lock.json
   
   # 重新安装
   npm install
   
   # 启动
   npm run dev
   ```

### 方案3: 使用预构建版本

如果开发服务器实在无法启动,可以构建生产版本:

```bash
cd "c:\Users\user17\Desktop\新建文件夹\日语\git words\jlpt-n1-study-new"

# 构建
npm run build

# 使用任意HTTP服务器运行dist文件夹
# 例如使用Python:
cd dist
python -m http.server 8080

# 或使用Node的serve包:
npx serve dist
```

### 方案4: 使用在线IDE

将项目上传到:
- CodeSandbox (https://codesandbox.io)
- StackBlitz (https://stackblitz.com)
- GitHub Codespaces

这些平台提供预配置的环境。

## 项目文件说明

### 核心文件
- `src/App.tsx` - 主应用组件

- `src/store/appStore.ts` - 状态管理
- `src/data/words.json` - 单词数据
- `src/data/grammar.json` - 语法数据

### 配置文件
- `vite.config.ts` - Vite配置
- `tailwind.config.js` - TailwindCSS配置
